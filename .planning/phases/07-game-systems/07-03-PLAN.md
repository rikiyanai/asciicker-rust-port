## WorldInstance Entity Spawner Guidance (ECS-FIX-04)

**Source:** Existing Code Audit RISK-1 — `WorldInstance` enum in `asset_loader/a3d_world.rs` currently has variants `Mesh`, `Sprite`, `Item` as plain Rust data. When Phase 7+ spawns these as Bevy entities, the enum-to-entity mapping needs careful design.

**Pattern:** Keep `WorldInstance` as a plain Rust enum for parsing (it is a data-loading type, not an ECS type). Create separate spawner functions for each variant that map to specific entity archetypes with marker components:

```rust
// Marker components for world instance types (following BL-2 enum component pattern)
#[derive(Component)]
pub enum WorldInstanceKind {
    Mesh,
    Sprite,
    Item,
}

// Spawner functions — one per instance type
pub fn spawn_mesh_instance(commands: &mut Commands, mesh: &MeshInstanceData) -> Entity {
    commands.spawn((
        WorldInstanceKind::Mesh,
        Transform::from_translation(mesh.position),
        // mesh-specific components...
    )).id()
}

pub fn spawn_sprite_instance(commands: &mut Commands, sprite: &SpriteInstanceData) -> Entity {
    commands.spawn((
        WorldInstanceKind::Sprite,
        Transform::from_translation(sprite.position),
        // sprite-specific components...
    )).id()
}

pub fn spawn_item_instance(commands: &mut Commands, item: &ItemInstanceData) -> Entity {
    commands.spawn((
        WorldInstanceKind::Item,
        Transform::from_translation(item.position),
        // item-specific components...
    )).id()
}
```

**Key decisions:**
1. Use an `enum WorldInstanceKind` component (BL-2 pattern), NOT separate marker structs per type. This enables exhaustive `match` in systems and prevents query mismatches when a new variant is added.
2. Parsing code (`a3d_world.rs`) remains pure Rust with no ECS coupling. The spawner functions live in a separate module (e.g., `world/spawners.rs`).
3. Systems that process world instances use `With<WorldInstanceKind>` to avoid matching character entities, and can further filter with enum matching.

**Timing:** This is NOT implemented in Phase 7 Plan 03 (networking only). It is guidance for whichever future plan introduces world instance entity spawning. The spawner function pattern precedent is established by this plan's direct `spawn_character_world()` call in `handle_player_connect` (R14-F147).
