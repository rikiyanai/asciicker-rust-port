2. **Create WGSL shader** at `engine-port/src/output/shader.wgsl`. Adapt from Mage Core shader.wgsl:
   - Fragment function ONLY (Bevy's fullscreen_vertex_shader provides the vertex stage).
   - Import Bevy's fullscreen vertex output via `#import bevy_core_pipeline::fullscreen_vertex_shader::FullscreenVertexOutput`.
   - Bindings: group(0) binding(0-3) = t_fore, t_back, t_text, t_font as `texture_2d<f32>`. group(1) binding(0) = uniforms with font_width: u32, font_height: u32.
   - Fragment logic: compute `p = vec2(pos.x - 0.5, pos.y - 0.5)` for pixel center. Compute cell coords `cp = vec2(i32(p.x) / i32(fw), i32(p.y) / i32(fh))` and local pixel `lp = vec2(i32(p.x) % i32(fw), i32(p.y) % i32(fh))`. Load fore/back/text at cp. Decode char index `c = i32(text.r * 255.0)`. Compute font atlas coords `fx = c % 16`, `fy = c / 16`, `lx = fx * i32(fw) + lp.x`, `ly = fy * i32(fh) + lp.y`. Sample font pixel. Return fore if font_pixel.r >= 0.5, else back.
   - CRITICAL: i32 casts for integer division/modulo, `pos.x - 0.5` pixel center adjustment.

3. **Create gpu_types.rs** at `engine-port/src/output/gpu_types.rs`:
   - `AsciiUniforms`: `#[repr(C)] #[derive(Copy, Clone, Debug, bytemuck::Pod, bytemuck::Zeroable)]` with `font_width: u32`, `font_height: u32`, `_padding: [u32; 2]` (16-byte GPU alignment).
   - `AsciiRenderConfig`: `#[derive(Resource)]` with `font_width: u32`, `font_height: u32`, `font_atlas_handle: Handle<Image>`. Lives in Main World.
   - `ExtractedAsciiGrid`: `#[derive(Resource)]` with `width: u32`, `height: u32`, `char_data: Vec<u8>`, `fg_data: Vec<u8>`, `bg_data: Vec<u8>`, `font_width: u32`, `font_height: u32`.
   - `pub fn extract_grid_data(grid: &AsciiCellGrid, config: &AsciiRenderConfig) -> ExtractedAsciiGrid` that converts char_indices (u16) to Rgba8 bytes (R=idx as u8, G=0, B=0, A=255), flattens fg_colors and bg_colors to flat Vec<u8>, copies dimensions and font sizes.
   - **XP-040 FIX:** WARNING: `char_indices` is `Vec<u16>` but the GPU encoding truncates to `u8` (R channel of the Rgba8Unorm texture). Values >255 will be silently truncated. Current CP437 glyphs are 0-255 (safe). The `debug_assert!(idx <= 255)` added in Phase 3.1 catches this in debug mode only; release builds will silently truncate without the assert.
   - **P3-M01 FIX:** The phrase "added in Phase 3.1" above is a forward reference. The `debug_assert!(idx <= 255)` guard was introduced in Phase 3.1 (031-01-PLAN.md AUDIT-04). It is NOT part of Phase 3 deliverables. This plan text anticipates it retroactively; the assert does not exist until Phase 3.1 executes.
   - Unit tests: 2x2 grid extraction verifies char_data encoding (4 bytes per cell), fg/bg flattening, dimensions match.
