  <action>
Create `engine-port/src/output/gpu_plugin.rs` implementing the full Bevy render pipeline. This is a single complex file following Bevy's `custom_post_processing` example pattern. Key components:

**1. Shader Handle:**
- Use `embedded_asset!` macro or `load_internal_asset!` to embed shader.wgsl. The shader handle must be available when creating the render pipeline.
- Alternative: Use `AssetServer::load("embedded://asciicker_engine/output/shader.wgsl")` with `embedded_asset!(app, "shader.wgsl")` in plugin build.

**2. AsciiGpuPlugin (implements Plugin):**
- In `build()`:
  - Call `embedded_asset!(app, "shader.wgsl")` to register the shader.
  - Get the RenderApp sub-app via `app.sub_app_mut(RenderApp)`.
  - Add extract system: `extract_ascii_grid` in `ExtractSchedule`.
  - Add prepare system: `prepare_ascii_textures` in `Render` schedule at `RenderSet::Prepare`.
  - Add render graph node: Create `AsciiNode` label, add node to `Core2d` subgraph after `Node2d::MainPass`.
- **[SUPERSEDED]** ~~In `finish()`: Create the `AsciiPipeline` resource in the render world (pipeline cache, bind group layouts, sampler, font atlas handle).~~ Implementation uses `RenderStartup` schedule instead of `Plugin::finish()`. See 03-02-SUMMARY.md for the actual approach used. `AsciiPipeline` is created via the `init_ascii_pipeline` system added to the `RenderStartup` schedule.

**3. Extract System (`extract_ascii_grid`):**
- System params: `grid: Extract<Res<AsciiCellGrid>>`, `config: Extract<Res<AsciiRenderConfig>>`, `mut commands: Commands`.
- Unconditionally call `extract_grid_data(&grid, &config)` from gpu_types.rs.
- Insert the `ExtractedAsciiGrid` as a resource in the render world via `commands.insert_resource(extracted)`.
- This runs every frame regardless of changes (GPU-04 requirement: unconditional extraction).

**4. AsciiPipeline Resource:**
- `#[derive(Resource)]` struct holding: `pipeline_id: CachedRenderPipelineId`, `texture_bind_group_layout: BindGroupLayout`, `uniform_bind_group_layout: BindGroupLayout`, `sampler: Sampler` (nearest, no filtering), `font_atlas_handle: Handle<Image>`.
- **P3-H02 FIX:** IMPLEMENTATION NOTE: The actual `AsciiPipeline` struct stores `BindGroupLayoutDescriptor` (not compiled `BindGroupLayout`), has no `sampler` field (nearest-filter is implicit in `TextureSampleType::Float { filterable: false }`), and has no `font_atlas_handle` field (the handle travels via `ExtractedFontAtlasHandle` resource extracted each frame). The existing **P3-002 FIX** note covers the `finish()` supersession; this note documents the field-level divergence.
- Created in `finish()` using the render world's `RenderDevice`:
  - Define texture_bind_group_layout with 4 entries (binding 0-3, texture_2d, float non-filterable, fragment visibility).
  - Define uniform_bind_group_layout with 1 entry (binding 0, uniform buffer, fragment visibility).
  - Create render pipeline descriptor: vertex=fullscreen_vertex_shader (use `FullscreenShader` from bevy_core_pipeline), fragment=ascii shader with the two bind group layouts. Format=`TextureFormat::bevy_default()`. Blend=REPLACE. Primitive=default (TriangleList, Bevy fullscreen uses vertex index trick).
  - **P3-001 FIX:** "`blend: None`" means no blending (full opaque overwrite), NOT the named `BlendState::REPLACE` constant. The value `None` in the pipeline descriptor sets the blend state to disabled (opaque overwrite), whereas the string "REPLACE" above is informal shorthand — not a Bevy constant reference.
  - Submit to PipelineCache, store CachedRenderPipelineId.

**5. AsciiGpuTextures Resource:**
- `#[derive(Resource, Default)]` struct holding: `char_texture: Option<Texture>`, `fg_texture: Option<Texture>`, `bg_texture: Option<Texture>`, `uniform_buffer: Option<Buffer>`, `bind_group: Option<BindGroup>`, `last_width: u32`, `last_height: u32`.
- Stores GPU textures and bind group. Persists across frames as a render-world Resource.

**6. Prepare System (`prepare_ascii_textures`):**
- System params: `extracted: Res<ExtractedAsciiGrid>`, `pipeline: Res<AsciiPipeline>`, `mut textures: ResMut<AsciiGpuTextures>`, `render_device: Res<RenderDevice>`, `render_queue: Res<RenderQueue>`, `gpu_images: Res<RenderAssets<GpuImage>>`.
- Check if font atlas GpuImage is ready: `gpu_images.get(&pipeline.font_atlas_handle)`. If None, return early (Pitfall 4: font not loaded yet).
- If grid dimensions changed (width != last_width or height != last_height), recreate all 3 data textures with new size. Create them via `render_device.create_texture()` with Rgba8Unorm format, COPY_DST | TEXTURE_BINDING usage. Update last_width/last_height.
- Create uniform buffer with AsciiUniforms (font_width, font_height, padding). Can recreate each frame or create once and reuse.
- Upload texture data: `render_queue.write_texture()` for char_data, fg_data, bg_data from ExtractedAsciiGrid.
- Create bind group: group(0) = 4 texture views (fg, bg, char, font atlas from GpuImage), group(1) = uniform buffer. **I-02 NOTE (Round 4):** Binding order must match shader declarations exactly: binding(0)=t_fore (fg), binding(1)=t_back (bg), binding(2)=t_text (char), binding(3)=t_font (font atlas). Constructed as [fore_view, back_view, text_view, font_view] in that order.
- Store bind group in AsciiGpuTextures resource.
- CRITICAL: Texture format MUST be `TextureFormat::Rgba8Unorm` (NOT sRGB) for all data textures (Pitfall 1).
- CRITICAL: Font atlas texture view comes from `gpu_images.get(&handle).unwrap().texture_view` -- Bevy manages the GPU upload.
- For the font atlas, force linear format by setting `is_srgb: false` on the Image asset via `ImageLoaderSettings` in AssetServer::load_with_settings (update Plan 01's AsciiRenderConfig setup if needed, or handle it here).

**7. AsciiNode (ViewNode):**
- Implement `ViewNode` trait for `AsciiNode`. The `Node::run` method:
  - Get `AsciiGpuTextures` and `AsciiPipeline` from render world.
  - If bind_group is None, return Ok(()) (font not loaded yet, skip rendering).
  - Get cached pipeline from PipelineCache. If not ready, return Ok(()).
  - Begin render pass on the view target (from `graph.view_target()`). Clear color black.
  - Set pipeline, set bind groups (0 = textures, 1 = uniforms).
  - Draw 3 vertices (fullscreen triangle, Bevy convention).
  - End render pass, return Ok(()).

**8. Render Graph Wiring:**
- Define `AsciiNodeLabel` as a render graph label (use `#[derive(Debug, Hash, PartialEq, Eq, Clone, RenderLabel)]`).
- In plugin build, add node to Core2d: `render_app.add_render_graph_node::<ViewNodeRunner<AsciiNode>>(Core2d, AsciiNodeLabel)`.
- Add edge: `render_app.add_render_graph_edge(Core2d, Node2d::MainPass, AsciiNodeLabel)`.
  - **P3-009 FIX:** Implementation uses `Node2d::EndMainPass` (not `MainPass`). The ASCII output draws after Bevy's 2D main pass has fully completed. The plan text above says "after `Node2d::MainPass`" but the actual edge in implementation targets `Node2d::EndMainPass`.
  </action>
