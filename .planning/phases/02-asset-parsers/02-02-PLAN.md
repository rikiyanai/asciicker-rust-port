**GREEN:** Create `a3d_terrain.rs`:

Types:
- `FileHeader` -- `#[repr(C, packed)]` with `#[derive(Copy, Clone, bytemuck::Pod, bytemuck::Zeroable)]`: file_sign u32, header_size u32, num_patches u32, reserved u32.
- `FilePatch` -- `#[repr(C, packed)]` with Pod/Zeroable: x i32, y i32, visual [[u16; VISUAL_CELLS]; VISUAL_CELLS] (128 bytes), height [[u16; (HEIGHT_CELLS+1)]; (HEIGHT_CELLS+1)] (50 bytes), diag u16. Static assert size == 188.
- `TerrainPatch` -- owned Rust struct (not packed): x i32, y i32, visual [[u16; 8]; 8], height [[u16; 5]; 5], diag u16.
- `A3dTerrain` -- `patches: Vec<TerrainPatch>`.
- `MatCell` -- fg [u8; 3], gl u8, bg [u8; 3], flags u8 (8 bytes).
- `MaterialTable` -- `materials: Vec<[[MatCell; 16]; 4]>` (256 entries, 4 elevations, 16 diffuse).

Functions:
- `pub fn parse_terrain_section(data: &[u8]) -> Result<(A3dTerrain, usize), AssetError>` -- returns terrain + bytes consumed.
  1. Read FileHeader via `bytemuck::from_bytes(&data[0..16])`.
  2. Validate file_sign == A3D_MAGIC, header_size == 16.
  3. Read num_patches FilePatch structs at offset 16, each 188 bytes.
  4. Convert packed FilePatch to owned TerrainPatch (copy data out of packed struct).
  5. Return consumed = 16 + num_patches * 188.

- `pub fn parse_material_section(data: &[u8]) -> Result<(MaterialTable, usize), AssetError>` -- reads 131,072 bytes.
  1. Validate data.len() >= MATERIAL_TABLE_SIZE.
  2. Read 256 materials. Each material: 512 bytes = shade[4][16] where each MatCell is 8 bytes.
  3. Return consumed = MATERIAL_TABLE_SIZE.

CRITICAL: All integer reads use little-endian. bytemuck handles this for `#[repr(C)]` on LE platforms. Add a compile-time assertion: `#[cfg(not(target_endian = "little"))] compile_error!("A3D parsing requires little-endian platform");`

Run `cargo test --test a3d_terrain_parser` -- MUST PASS.
