**GREEN:** Create `xp_sprite.rs` with:
- Types: `XpCell { glyph: u32, fg: [u8; 3], bg: [u8; 3] }`, `XpLayer { width, height, cells: Vec<XpCell> }`, `XpSprite { version: i32, width, height, layers: Vec<XpLayer> }`.
- `MergedCell { glyph: u32, fg: [u8; 3], bg: [u8; 3], height: u8, transparent: bool }`.
- `pub fn parse_xp(bytes: &[u8]) -> Result<XpSprite, AssetError>`:
  1. GzDecoder decompress to Vec<u8>.
  2. Read 16-byte header: version (i32 LE), num_layers (i32 LE), width (i32 LE), height (i32 LE).
  3. Validate num_layers >= 3, width >= 1, height >= 1.
  4. For each layer: layer 0 starts at offset 16. Layer N>0: skip 8 bytes (per-layer w/h) before cell data. Read width*height cells column-major (col*height+row). Each cell: u32 LE glyph + 3 fg bytes + 3 bg bytes = 10 bytes.
  5. Return XpSprite.

- `pub fn merge_layers(sprite: &XpSprite) -> Vec<MergedCell>`:
  Implements layer merge logic per C++ sprite.cpp semantics:
  1. Start with layer 2 (visual) as base. Extract transparency from layer 0 bg color: cell is transparent if its bg matches layer 0's bg at same position.
  2. Extract height from layer 1 glyph ('0'-'9' -> 0-9, 'A'-'Z' -> 10-35, else -> SPRITE_HEIGHT_UNDEFINED).
  3. For layers 3..N-1 (intermediate, Rust exclusive upper bound — processes layers 3 through N-2 inclusive): simple overwrite -- if cell's bg != layer 0 colorkey bg, replace base cell.
  <!-- R7-011 NOTE (SUPERSEDED by R8-001): Original range `3..N-2` was incorrect — it skipped layer N-2 for 6+ layer sprites. Fixed to `3..N-1` (Rust exclusive upper bound), which processes layers 3 through N-2 inclusive. Layer N-1 remains the swoosh layer handled in step 4. For sprites with exactly 4 layers, range 3..3 is empty (only swoosh at layer 3=N-1). For 5+ layers, all intermediate layers between visual base (2) and swoosh (N-1) are covered. -->
  4. For layer N-1 (LAST layer only): swoosh merge detection. A cell is a swoosh cell if its fg == SPRITE_CYAN (0,255,255) AND its glyph is a half-block (220-223). For swoosh cells: use the half-block glyph mask (220=lower->MASK_LOWER, 221=left->MASK_LEFT, 222=right->MASK_RIGHT, 223=upper->MASK_UPPER) to determine which quadrants the swoosh affects. For affected quadrants, lighten the base cell's color by SPRITE_LIGHTEN_AMOUNT (clamp at 255). For non-swoosh cells on the last layer: simple overwrite (same as intermediate layers).
  <!-- **P2-201 FIX RESOLVED:** Swoosh detection uses glyphs 220-223 (excluding 219=SPRITE_GLYPH_FULL_BLOCK). C++ constants.rs defines SPRITE_GLYPH_FULL_BLOCK=219 as a separate non-swoosh constant. Range 220-223 is confirmed correct. -->
  5. Return merged cells with transparency, height, and final glyph/color data.

  NOTE: This is the "basic swoosh" implementation covering detection, half-block masking, and lighten. Full AverageGlyphTransp (sub-cell blending with partial transparency) is deferred to Phase 5 rendering integration where the palette quantization context is available. The merge_layers function handles the structural merge; Phase 5 will handle the palette-dependent glyph averaging.

CRITICAL pitfall avoidance: column-major order, 8-byte inter-layer gap, little-endian integers, swoosh ONLY on last layer (Pitfall 6).

Run `cargo test --test xp_parser` -- MUST PASS.
