1. **WaterReflectionStage** -- Port Stage 5 skeleton from render.cpp:3266-3374. This stage:
   - Takes water_z (water surface height) and the current view matrix
   - Flips Z in view matrix: Z' = 2*water_z - Z (reflection across water plane)
   - Sets a `reflected` flag (maps to C++ global_refl_mode)
   - Reflected samples get spare |= 0x3 (parity bits set, marking them as reflections)
   - Structure as a callable function: `render_water_reflections(sample_buffer: &mut SampleBuffer, water_z: f32, view_matrix: &[f64; 16])` -- implements Z-flip and spare-bit marking against existing SampleBuffer contents below water_z
   - NOT DONE: Full terrain/world geometry re-query is deferred until Phase 5 render pipeline is complete. This skeleton only flips Z and marks spare bits on already-rasterized samples.

2. **apply_water_ripple** -- Port Perlin ripple from render.cpp:3860-3903. In the resolve pass, for cells where spare & 0x3 == 3 (reflected cells):
   - Use `noise::Fbm::<noise::Perlin>::new(0)` with 4 octaves
   - Sample: `d = fbm.get([wx as f64 * 0.05, wy as f64 * 0.05, time as f64])`
   - Normalize to [0,1]: `d_norm = (d + 1.0) * 0.5`
   - Compute color shift: `id = (d_norm * 5.0) as i32 - 2` clamped to [-2, 2]
   - Apply to xterm-256 fg color: shift by +/-(1 + 6 + 36) steps in the 6x6x6 RGB cube. id > 0 lightens, id < 0 darkens. Clamp to valid palette range [16, 231].
   - Function signature: `apply_water_ripple(cell: &mut AnsiCell, wx: f32, wy: f32, time: f32)`

Add `pub mod water;` to render/mod.rs. Update CpuRasterizerPlugin to store water_z as a resource or config field.
