**ActionState enum** ported from game.h ACTION, adding Block per CHAR-01 ("idle, walk, run, attack, block, dead" -- 6 states):

Variants: `None` (idle/walk/run -- speed determined by PhysicsIO velocity magnitude), `Attack`, `Block`, `Fall`, `Dead`, `Stand` (standing up from fall). Derive `Component, Default, Clone, Copy, PartialEq, Eq`. `#[default]` on `None`.

NOTE on Block: C++ has no ACTION_BLOCK enum. Blocking is handled via shield equipment state + input flag. Rust port adds explicit Block variant per CHAR-01. Block is movement-locked like Attack.

**`can_transition_to(&self, target) -> bool`** transition guards:
- None: always allowed
- Attack: blocked from Fall, Stand, Dead, Block
- Block: blocked from Fall, Stand, Dead, Attack
- Fall: blocked only from Dead. Enterable from None, Attack, Block, Stand.
- Stand: only from Fall or Dead
- Dead: from any state

The state machine checks state-level guards ONLY. Equipment guards (shield required for Block) are enforced by the CALLER in `input.rs`:
```rust

if sprite_req.shield != Shield::None && keyboard.just_pressed(KeyCode::KeyF) {
    action_state.try_transition(ActionState::Block);
}
```

**`try_transition(&mut self, target) -> bool`**: checks guard, applies if valid, returns success.

**`check_animation_complete(&mut self, anim: &AnimationState) -> bool`**: returns true when current action's animation has finished. Auto-transitions:
- Attack -> None (attack animation finished)
- Stand -> None (stand-up animation finished)
- Fall: does NOT auto-transition (stays until grounded)
- Block: does NOT auto-transition (stays until key released)
- Dead: does NOT auto-transition (permanent)

        Mount::None => (2.0_f32, 7.0_f32),
        Mount::Wolf | Mount::Bee => (3.0_f32, 9.0_f32),

    let world_radius = radius_cells / (3.0 * HEIGHT_CELLS as f32) * VISUAL_CELLS as f32;

**`advance(&mut self, action: ActionState, player_stp: i32, delta_us: u64)`** updates frame_index:

**R19-M05 FIX (MEDIUM):** C++ player_stp increment (physics.cpp:1481) uses
`(~(1 << 31)) & (player_stp + (int)(64 * xy_vel))` = 0x7FFFFFFF mask (sign-bit mask to
prevent overflow), NOT STEP_MASK. STEP_MASK (8*1024-1 = 8191) is used only for FRAME
EXTRACTION below. Plan 06-01's update_output_system must apply `& 0x7FFFFFFF` for the
increment, and this function applies `& STEP_MASK` only for frame extraction. Do NOT
conflate the two masks.

- None (walk, player_stp != -1): `frame = ((player_stp + STEP_OFFSET) & STEP_MASK) / STEP_DIVISOR` (8 frames)
- None (idle, player_stp == -1): `frame = 0`

- Block: `elapsed / BLOCK_US_PER_FRAME` (hold on last frame while blocking)
- Fall: `elapsed / FALL_US_PER_FRAME`
- Stand: `elapsed / STAND_US_PER_FRAME`, capped at STAND_FRAME_COUNT
- Dead: last frame (frozen)

When action changes from previous call, reset `elapsed_frames = 0`.

#### sprite_query.rs

```rust
pub fn query_character_sprites(
    characters: Query<(&SpriteReq, &AnimationState, &Transform), With<Character>>,
    camera: Res<GameCamera>,
    physics_io: Res<PhysicsIO>,
    mut sprite_queue: ResMut<SpriteQueue>,
) {
    for (sprite_req, anim_state, transform) in &characters {
        let pos = transform.translation;

        // 1. Sprite name from kind + action + equipment
        let sprite_name = format!(
            "{:?}_{:?}",
            sprite_req.kind, sprite_req.action
        ).to_lowercase();

        // 2. Animation frame
        // **R19-M02 FIX (CRITICAL):** SpriteRenderEntry.frame is u32, not i32.
        // SpriteRenderEntry.anim is also u32, not i32. Casting as i32 is a type error.
        let frame = anim_state.frame_index; // already u32, no cast needed

        // 3. Screen position: project through camera
        // **R19-M01 FIX (CRITICAL):** camera.view_tm is [f64; 16] (raw array), NOT a Mat4.
        // `camera.view_tm * world_pos` is a compile error -- [f64; 16] does not impl Mul<Vec4>.
        // The camera uses architectural perspective projection (not a standard MVP matrix).
        // Use project_world_to_screen() from pipeline.rs (lines 74-115) or the equivalent
        // transform_vertex_perspective() helper. These functions handle HEIGHT_CELLS scaling,
        // scene_shift, and the mul/add perspective parameters correctly.
        // Returns Option<(i32, i32)> matching SpriteRenderEntry's i32 screen coords.
        let pos_arr = [pos.x, pos.y, pos.z];
        let (screen_x, screen_y) = match project_world_to_screen(&pos_arr, &camera) {
            Some(coords) => coords,  // (i32, i32)
            None => continue,        // behind camera, skip this character
        };

        // 4. Distance for far-to-near sort (2D XY distance)
        let dist = (camera.pos[0] - pos.x).hypot(camera.pos[1] - pos.y);

        // 5. Yaw from PhysicsIO (authoritative value, NOT transform rotation)
        let yaw = physics_io.yaw;

        // **R19-M10 FIX (MEDIUM):** C++ uses an 8-direction index derived from the
        // camera-relative facing angle to select the correct sprite rotation frame.
        // The direction = ((entity_yaw - camera_yaw + 360 + 22.5) / 45.0) as u32 % 8.
        // SpriteRenderEntry stores `yaw` but blit_sprite needs a direction index (0-7)
        // for sprite sheet column lookup. Either compute direction here and store it
        // (requires adding a `direction: u32` field to SpriteRenderEntry), or compute
        // it inside blit_sprite from yaw + camera.yaw. Document this gap if not
        // implementing now -- the placeholder 'S' blit doesn't use direction anyway.

        let entry = SpriteRenderEntry {
            dist,
            screen_x,  // R19-M02: already i32 from project_world_to_screen
            screen_y,  // R19-M02: already i32 from project_world_to_screen
            sprite_name,
            pos: [pos.x, pos.y, pos.z],
            yaw,
            anim: sprite_req.action as u32,  // R19-M02 FIX: field is u32, not i32
            frame,                            // R19-M02 FIX: field is u32, no cast needed
        };
        sprite_queue.push(entry);
        // NOTE: Verify SpriteQueue API at execution time. If SpriteQueue is
        // a newtype Vec, use sprite_queue.0.push(entry) instead.
    }
}
```

pub fn update_character_state(
    physics_io: Res<PhysicsIO>,
    mut q: Query<(&mut ActionState, &AnimationState, &SpriteReq), With<Character>>,
) {
    for (mut action_state, anim_state, _sprite_req) in &mut q {
        // Auto-transition on animation complete (Attack->None, Stand->None)
        action_state.check_animation_complete(anim_state);

        // Fall detection: not grounded AND moving downward
        if !physics_io.grounded && physics_io.vel_z < -Z_THRESH {
            action_state.try_transition(ActionState::Fall);
        }
        // Jump arc (vel_z > 0, not grounded) does NOT enter Fall

        // Without this, a character that enters Fall state stays in Fall forever after
        // landing (grounded becomes true but nothing transitions out of Fall).
        // Add: if action == Fall && physics_io.grounded -> try_transition(Stand)
        if *action_state == ActionState::Fall && physics_io.grounded {
            action_state.try_transition(ActionState::Stand);
        }
    }
}
