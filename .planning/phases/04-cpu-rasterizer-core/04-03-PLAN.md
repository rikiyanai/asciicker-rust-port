  **Bresenham function** (port of C++ render.cpp:111-184):
  ```rust
  pub fn bresenham(
      buf: &mut [Sample],
      w: i32, h: i32,
      from: [i32; 3],  // [x, y, z] in sample-buffer coords
      to: [i32; 3],
      or_bits: u8,      // spare bits to OR (e.g., 0x04 for grid)
  )
  ```
  - Compute sx, sy, sz (deltas). Early return if sx==0 && sy==0.
  - Compute ax = sx.abs(), ay = sy.abs().
  - Swap from/to so traversal goes in positive major-axis direction.
  - If ax >= ay (horizontal domain):
    - Step by 2 (due to 2x supersampling): x0 = max(0, from[0]) rounded up to even, x1 = min(w, to[0])
    - For each x in x0..x1 step 2: compute y via linear interpolation, z via linear interpolation
    - If y in [0, h): write `buf[w*y + x].spare |= or_bits` after depth_test_ro check, also write buf[w*y + x + 1]
  - If ay > ax (vertical domain):
    - Step by 1: y0 = max(0, from[1]), y1 = min(h, to[1])
    - For each y: compute x via linear interpolation, z via interpolation
    - If x in [0, w): write `buf[w*y + x].spare |= or_bits` after depth_test_ro check
  - Use `1.0 / delta as f32` for interpolation factor (NOT integer division)
  - **P4-L01 FIX:** Clarify which delta to use per mode: for horizontal mode use `let n = 1.0_f32 / sx as f32`; for vertical mode use `let n = 1.0_f32 / sy as f32`. Note that `as` binds tighter than `/` in Rust, so `1.0 / delta as f32` is equivalent to `1.0 / (delta as f32)` — the precedence is correct, but using the explicit form `1.0_f32 / sx as f32` avoids ambiguity.
  - Use `.floor() as i32` for coordinate rounding (matches C++ cast behavior)

  **Module registration** (engine-port/src/render/mod.rs):
  - Add `pub mod rasterizer;`

  **Tests:**
  - Horizontal line from (0,5,100) to (20,5,100) with or_bits=0x04: samples at even x positions along y=5 have spare & 0x04 set

  <name>Task 2: Implement barycentric triangle rasterizer with RasterShader trait</name>
  <files>engine-port/src/render/rasterizer.rs</files>
  <action>
  Add to engine-port/src/render/rasterizer.rs:

  **RasterShader trait:**
  ```rust
  pub trait RasterShader {
      fn blend(&self, sample: &mut Sample, z: f32, bc: [f32; 3]);
  }
  ```
  - Use `impl RasterShader` (NOT `dyn RasterShader`) in all function signatures for zero-cost static dispatch matching C++ template inlining.

  **Helper functions** (private):
  - `bc_a(a: &[i32; 4], b: &[i32; 4], c: &[i32; 4]) -> i32`: 2 * signed area = `2 * ((b[0]-a[0])*(c[1]-a[1]) - (b[1]-a[1])*(c[0]-a[0]))`
  - `bc_p(a: &[i32; 4], b: &[i32; 4], cx: i32, cy: i32) -> i32`: `(b[0]-a[0])*(2*cy+1-2*a[1]) - (b[1]-a[1])*(2*cx+1-2*a[0])`

  **rasterize function** (port of C++ render.cpp:404-557):
  ```rust
  pub fn rasterize(
      buf: &mut [Sample],
      w: i32, h: i32,
      shader: &impl RasterShader,
      v: [&[i32; 4]; 3],  // 3 vertices, each [x, y, z, cull_flags]
      double_sided: bool,
  )
  ```
  - Cull check: if `v[0][3] & v[1][3] & v[2][3] != 0` return (common frustum bit)
  - Compute area = bc_a(v[0], v[1], v[2])
  - If area > 0: CCW winding. If area >= 0x10000: degenerate, return. Call rasterize_inner with positive area.
  - If area < 0 && double_sided: CW winding. If area <= -0x10000: degenerate, return. Call rasterize_inner with negative area (negate barycentrics).
  - If area == 0: degenerate, return.

  **rasterize_inner** (private, extracted from rasterize):
  - Compute bounding box of triangle, clamp to [0, w) x [0, h)
  - Precompute 1/area as f32 for barycentric normalization
  - For each pixel (x, y) in bounding box:
    - Compute bc0 = bc_p(v[1], v[2], x, y), bc1 = bc_p(v[2], v[0], x, y), bc2 = bc_p(v[0], v[1], x, y)
    - Edge test: all bc >= 0 (for CCW). Apply tie-breaking from C++ render.cpp:478-483:
      - If bc0 == 0: include only if v[1][0] < v[2][0] (or v[1][0]==v[2][0] && v[1][1] < v[2][1])
      - Same logic for bc1 (comparing v[2] vs v[0]) and bc2 (comparing v[0] vs v[1])
    - Normalize: bc_f = [bc0, bc1, bc2] / area as f32
    - Interpolate z: z = bc_f[0]*v[0][2] + bc_f[1]*v[1][2] + bc_f[2]*v[2][2]
    - Call shader.blend(&mut buf[w*y + x], z as f32, bc_f)

  **Test shader** (in tests module):
  ```rust
  struct FlatShader { visual: u16, diffuse: u8, spare: u8 }
  impl RasterShader for FlatShader {
      fn blend(&self, sample: &mut Sample, z: f32, _bc: [f32; 3]) {
          // R6-L03 NOTE: This test shader intentionally uses a stricter depth test (sample.height > z)
          // instead of sample.depth_test_ro(z) for test predictability. The depth_test_ro tolerance
          // path (height <= z + 8.0) is covered separately in material path tests.
          if sample.height > z || sample.height == Sample::CLEAR_HEIGHT {
              sample.visual = self.visual;
              sample.diffuse = self.diffuse;
              sample.spare = self.spare;
              sample.height = z;
          }
      }
  }
  ```

  **Tests:**
  - Small triangle (3 vertices at known positions): verify samples inside triangle have shader-written values
  - Samples outside triangle remain at clear state
  - Double-sided triangle: both CW and CCW winding produce filled pixels
  - Single-sided triangle: CW winding is culled (no pixels written)
  - Degenerate triangle (collinear vertices, area=0): no pixels written
  - Frustum-culled triangle (all vertices share cull bit): no pixels written
  - Depth test: closer triangle overwrites farther one's samples
  - Two adjacent triangles sharing an edge: no double-draw on the shared edge (tie-breaking works)
