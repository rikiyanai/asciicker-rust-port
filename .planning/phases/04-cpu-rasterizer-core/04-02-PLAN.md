<objective>
Implement the material system: MatCell/Material structs and the auto_mat LUT that maps RGB555 colors to xterm-256 palette pairs with dither glyphs.

Purpose: The resolve stage uses two code paths: terrain samples look up materials via `matlib[sample.visual].shade[elevation][diffuse]`, while mesh samples use `auto_mat[rgb555]` to get {bg, fg, dither_glyph}. Both systems must exist before the resolve stage can be built. This plan implements the data types and LUT generation.

Output: MatCell and Material structs, auto_mat LUT generator (98KB table), and a small set of hardcoded test materials for Phase 4 testing (full material library from .a3d files wired in Phase 5).
</objective>

  - `pub fn create_auto_mat() -> Box<[u8; 32 * 32 * 32 * 3]>`: use Box to avoid 98KB on stack
  - Use `std::sync::LazyLock` for a global static: `pub static AUTO_MAT: LazyLock<Box<[u8; 98304]>> = LazyLock::new(|| Box::new(create_auto_mat_inner()));`
  - **P4-I01 FIX (INFO):** The LUT size is stated as "98,304 bytes (32*32*32*3)" in the `must_haves` truths and as `Box<[u8; 98304]>` inline here. The actual implementation uses an `AUTO_MAT_SIZE` constant for both the type annotation and the `LazyLock`. Prefer `AUTO_MAT_SIZE` constant references in all descriptions to avoid number drift.
  - Implement the exact algorithm from the research doc:
    1. Compute `flo[x] = (5 * x) / 31` and `rem[x] = 5*x - 31*flo[x]` for x in 0..32
    2. Glyph array: `[b' ', b'.', b'.', b':', b':', b'%']`
    3. For each RGB555 (r,g,b in 0..32): find best pair of 8 cube vertices by minimizing perpendicular distance
    4. Compute projection (0.0 to 1.0) along best edge -> shd (0..11)
    5. If shd < 6: mat = [palette(lo), palette(hi), glyph[shd]], else swap and use glyph[11-shd]
    6. `mcv_to_5(mcv) = (mcv * 5 + 2) / 5` converts MCV-space (0-5) to palette-space (0-5)
    7. `palette_idx(vert) = 16 + 36*r5 + 6*g5 + b5` using mcv_to_5 on each component
  - Follow the EXACT code from 04-RESEARCH.md auto_mat section. Do NOT simplify the math.

  **Accessor function:**
  - `pub fn auto_mat_lookup(rgb555: u16) -> (u8, u8, u8)`: extracts r5, g5, b5 from rgb555, indexes AUTO_MAT, returns (bg_palette, fg_palette, dither_glyph)
  - **R6-M05 FIX:** Unpacking order: `r5 = rgb555 & 0x1F, g5 = (rgb555 >> 5) & 0x1F, b5 = (rgb555 >> 10) & 0x1F`. Index formula: `idx = 3 * (r5 + 32*g5 + 32*32*b5)`. Returns `(AUTO_MAT[idx], AUTO_MAT[idx+1], AUTO_MAT[idx+2])` as `(bg_palette, fg_palette, dither_glyph)`. Test: `auto_mat_lookup(0u16)` (pure black r5=g5=b5=0) → both palette indices = 16 (xterm black).

  - auto_mat_lookup for pure white (31,31,31) returns palette indices near 231 (white in xterm-256)
  - auto_mat_lookup for mid-grey (16,16,16): bg and fg should be adjacent palette colors

  - Verify symmetry: for any entry, bg and fg are always valid xterm-256 indices (16-231)
