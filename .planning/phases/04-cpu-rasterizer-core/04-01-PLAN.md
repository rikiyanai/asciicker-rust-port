  - **P4-M01 FIX:** IMPLEMENTATION NOTE: The `supersample_factor` field was absent from the Phase 1 `RenderConfig` stub — no removal was needed. The Consistency Diff claim below that "RenderConfig loses supersample_factor field" is inaccurate; the field never existed in the executed implementation.

  - Fields: `fg: u8` (xterm-256 palette index), `bk: u8` (xterm-256 palette index), `gl: u8` (CP437 glyph code, 255 = transparent), `spare: u8` (flags, 0xFF = rendered cell)

  **Color quantization** (engine-port/src/render/quantize.rs -- NEW FILE):
  - `rgb8_to_rgb5(c8: u8) -> u8`: `((c8 as u32 * 249 + 1014) >> 11) as u8` -- exact C++ formula for mesh shader
  - `rgb5_to_rgb8(c5: u16) -> u8`: `((c5 * 527 + 23) >> 6) as u8` -- exact C++ resolve expansion
  - `pack_rgb555(r5: u8, g5: u8, b5: u8) -> u16`: `r5 as u16 | ((g5 as u16) << 5) | ((b5 as u16) << 10)`
  - `unpack_rgb555(rgb555: u16) -> (u8, u8, u8)`: extract r5, g5, b5 from packed u16
  - `rgb888_to_rgb555(r: u8, g: u8, b: u8) -> u16`: convenience combining rgb8_to_rgb5 + pack_rgb555
  - `rgb555_to_rgb888(rgb555: u16) -> (u8, u8, u8)`: convenience combining unpack_rgb555 + rgb5_to_rgb8
  - `rgb2pal(rgb: [u8; 3]) -> u8`: `16 + 36 * ((r + 25) / 51) + 6 * ((g + 25) / 51) + ((b + 25) / 51)` -- xterm-256 6x6x6 cube
