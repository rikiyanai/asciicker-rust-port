  **resolve function** (port of C++ render.cpp:3412-3938, simplified for Phase 4):
  ```rust
  pub fn resolve(
      samples: &[Sample],
      dw: i32,              // sample buffer width (2*ascii_w + 4)
      dh: i32,              // sample buffer height (2*ascii_h + 4)
      ascii_w: i32,         // output width
      ascii_h: i32,         // output height
      materials: &[Material], // material library
      output: &mut [AnsiCell], // ascii_w * ascii_h output cells
  )
  ```

  **Per-cell logic** (for each output cell at (cx, cy)):
  1. Compute sample buffer position: `sx = 2 + 2*cx`, `sy = 2 + 2*cy` (skip the +2 border)
  2. Read 2x2 block: samples at `[sy*dw+sx, sy*dw+sx+1, (sy+1)*dw+sx, (sy+1)*dw+sx+1]`
  3. Average the 4 samples' height (ignoring CLEAR_HEIGHT samples), average diffuse
  4. Collect spare flags: OR of all 4 samples' spare bits
  5. Determine if mesh or material: `combined_spare & MESH_FLAG != 0` -> mesh path, else material path

  **Material path** (spare & 0x8 == 0):
  - Take the dominant visual (material index) from the 4 samples (use first non-clear sample)
  - Compute elevation (0-3) from row above: read `samples[(sy-1)*dw + sx]` height, compare to current block average height. If current > above: elevation increases. Map height difference to 0-3 range.
  - Compute diffuse level: `avg_diffuse / 17` clamped to 0-15
  - Look up `materials[visual as usize].shade[elevation][diffuse_level]` -> MatCell
  - Convert MatCell.fg and MatCell.bg via rgb2pal -> fg_palette, bg_palette
  - Use MatCell.gl as the glyph
  - Write AnsiCell { fg: fg_palette, bk: bg_palette, gl, spare: 0xFF }

  **Mesh path** (spare & 0x8 != 0):
  - Take the dominant visual (RGB555) from the 4 samples
  - Apply diffuse scaling: expand RGB555 to RGB888 via rgb5_to_rgb8, multiply by avg_diffuse/255, convert back to RGB555 via rgb8_to_rgb5
  - Look up auto_mat_lookup(scaled_rgb555) -> (bg, fg, dither_glyph)
  - Write AnsiCell { fg, bk: bg, gl: dither_glyph, spare: 0xFF }

  **Reflection path** (spare & PARITY_MASK == REFLECTION):
  - Same as mesh/material but use diffuse divisor 400 instead of 255 (darker reflections, TRAP-R03)
  - For Phase 4: implement the check but the actual reflection samples will only appear in Phase 5

  **Grid/wireframe overlay:**
  - If combined_spare & GRID (0x04): override glyph with grid character (b'+' for intersection, b'-' for horizontal, b'|' for vertical -- simplified for Phase 4)
  - If combined_spare & WIREFRAME (0x40): override glyph with wireframe character

  **Clear cells** (all 4 samples at CLEAR_HEIGHT):
  - Write AnsiCell { fg: 0, bk: 0, gl: b' ', spare: 0 }

  **Module registration** (engine-port/src/render/mod.rs):
  - Add `pub mod resolve;`

  **Tests:**
  - Resolve of all-clear SampleBuffer produces all-space AnsiCells
  - Resolve with a single terrain-material sample: correct MatCell lookup, correct palette conversion
  - Resolve with a single mesh RGB555 sample: correct auto_mat lookup, correct dither glyph
  - Resolve with grid line (spare & 0x04): glyph overridden to grid character
  - Resolve with mixed terrain/mesh in 2x2 block: mesh flag dominates (matches C++ behavior)
  - Elevation detection: sample with height > row-above height produces elevation > 0
  - Output dimensions: ascii_w * ascii_h AnsiCells produced
