    Define `PlayerInputMsg` struct.

    Includes: `x_force`, `y_force`, `jump`, `fly`, `yaw`.
    Register for 30Hz client-to-server transmission.
  </task>
  <task id="12.4.2" title="Implement Authoritative Server Physics" est="M">
    Implement `server_movement_system`.
    Requirement: FNET-04.
    Applies received client inputs to the authoritative `PhysicsIO` Resource (for player) or Component (for NPCs).
    Executes fixed-timestep physics ticks.
  </task>
  <task id="12.4.3" title="Implement Client-Side Prediction" est="L">
    Implement `client_prediction_system`.
    Requirement: FNET-04.
    Runs physics locally each frame using the same logic as the server.
    Stores a buffer of inputs and predicted positions.
  </task>
  <task id="12.4.4" title="Implement Server Reconciliation (Soft-Snapping)" est="M">
    When a server snapshot arrives, compare predicted position with authoritative position.
    Requirement: FNET-04.
    If error exceeds threshold, implement 'Soft Reconciliation':
    1. Update the authoritative base state in the prediction buffer.
    2. Instead of hard teleporting, apply a 'visual correction' lerp that smoothly pulls the client entity toward the server position over ~100ms.
    3. Re-run prediction for all inputs in the buffer starting from the new server-correct state.
  </task>
</tasks>
