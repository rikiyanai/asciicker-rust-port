- `GameCamera::extract_frustum_planes(&mut self, dw: f64, dh: f64)`:
  - Extract 4-6 frustum planes from the view-projection matrix
  - Each plane [a,b,c,d] normalized such that ax+by+cz+d >= 0 means inside
  - For isometric: planes from screen bounds transformed to world space
  - For perspective: planes from camera position + view direction + FOV
  - **R51 FIX: Cite algorithm explicitly.** Use Gribb/Hartmann method: extract planes from rows of the combined view-projection matrix M. Left = M[3]+M[0], Right = M[3]-M[0], Bottom = M[3]+M[1], Top = M[3]-M[1], Near = M[3]+M[2], Far = M[3]-M[2]. Normalize each [a,b,c,d] by dividing by sqrt(a²+b²+c²). Reference: C++ render.cpp frustum setup at lines ~2990-3020, which uses screen-bounds approach for the isometric case.
  - **P5-056 FIX: WARNING:** Gribb/Hartmann method requires a clip-space view-projection matrix. `view_tm` is a 2D affine world-to-screen transform, NOT a perspective projection. For isometric mode, derive frustum planes from screen boundaries back-projected to world space using `inv(view_tm)`. For perspective mode, derive from `view_pos`, `view_dir`, and `focal`/screen dimensions.
  **P5-323 INFO:** No worked code example is provided for either the isometric or perspective frustum plane derivation. The R51 FIX references C++ render.cpp lines ~2990-3020 for the isometric screen-bounds approach, but that requires reading the C++ source. For isometric mode, the key idea is: transform the 4 screen corners `(0,0)`, `(dw,0)`, `(0,dh)`, `(dw,dh)` through `inv(view_tm)` to get world-space corners, then compute the 4 boundary planes from those corners. For perspective mode, build the frustum from `view_pos`, `view_dir`, and the FOV angle derived from `focal`. Consider sketching at least the isometric plane derivation in the implementation to reduce first-implementation risk.
  **R7-006 FIX:** Isometric frustum derivation sketch (4 steps):
  1. Compute 4 screen-space corners: (0,0), (dw,0), (0,dh), (dw,dh) where dw=2*ascii_w+4, dh=2*ascii_h+4
  2. Apply inv(view_tm) to each corner -> 4 world-space boundary points
  3. Compute 4 boundary planes: left = cross(bottom_left, top_left), right = cross(top_right, bottom_right), top = cross(top_left, top_right), bottom = cross(bottom_right, bottom_left)
  4. Normalize each plane to unit-length normal. Result: `[Plane; 4]` (no near/far needed for isometric).
  NOTE: view_tm is NOT a clip-space projection (P5-056 FIX), so Gribb/Hartmann cannot be applied directly.

  **R13-001 FIX (PERSPECTIVE FRUSTUM DERIVATION SKETCH):**
  For perspective mode, derive frustum planes from view_pos, view_dir, and focal/screen dimensions:
  1. Compute `half_w = (sample_width as f64) * 0.5`, `half_h = (sample_height as f64) * 0.5`
  2. Derive FOV half-angles from focal: `half_angle_h = atan2(half_w, focal)`, `half_angle_v = atan2(half_h, focal)`
  3. Build `right` and `up` vectors from view_tm columns (right = column 0, up = column 1)
  4. Construct 4 frustum planes from `view_pos + view_dir +/- right*tan(half_angle_h) +/- up*tan(half_angle_v)`:
     - Left plane: normal = normalize(view_dir + right * tan(half_angle_h)), passes through view_pos
     - Right plane: normal = normalize(view_dir - right * tan(half_angle_h)), passes through view_pos
     - Top plane: normal = normalize(view_dir + up * tan(half_angle_v)), passes through view_pos
     - Bottom plane: normal = normalize(view_dir - up * tan(half_angle_v)), passes through view_pos
  5. No near/far planes (same as isometric)
