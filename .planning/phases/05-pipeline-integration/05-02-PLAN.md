<objective>
Build the BSP tree runtime system that converts parsed A3dWorld instances into a spatial BSP tree with frustum-culled traversal dispatching mesh and sprite callbacks.

Purpose: The BSP tree is the spatial acceleration structure for world geometry. Frustum-culled BSP traversal during the WORLD stage of the pipeline determines which mesh and sprite instances are visible, enabling efficient rendering of large worlds.

Output: `RuntimeWorld` resource with BSP tree built via SAH from parsed instances, frustum query dispatching mesh/sprite callbacks, and instance flags controlling visibility.
</objective>

Create `bsp.rs` with:
- `BspNode` enum matching C++ types:
  ```rust
  pub enum BspNode {
      Node {
          children: [Option<Box<BspNode>>; 2],
          bbox: [f64; 6],
      },
      NodeShare {
          children: [Option<Box<BspNode>>; 2],
          bbox: [f64; 6],
          instances: Vec<InstanceId>,  // straddling instances
      },
      Leaf {
          bbox: [f64; 6],
          instances: Vec<InstanceId>,
      },
      Inst {
          bbox: [f64; 6],
          instance: InstanceId,
      },
  }
  ```
- Helper `compute_surface_area(instances: &[BspItem]) -> f64` that computes surface area of the combined AABB of a set of instances
- Helper `combined_bbox(items: &[BspItem]) -> [f64; 6]` that returns the union AABB
- `BspItem` struct: `{ id: InstanceId, bbox: [f64; 6], centroid: [f64; 3] }`
- `build_bsp(items: &mut [BspItem]) -> BspNode` function implementing SAH:
  1. If items.len() == 0: panic (should not happen, caller filters)
  2. If items.len() == 1: return `BspNode::Inst { bbox, instance }`
  3. If items.len() <= 4: return `BspNode::Leaf { bbox, instances }` (small leaf threshold)
  4. Try all 3 axes (x=0, y=1, z=2):
     - Sort items by centroid along axis
     - For each split position (1..len), compute left_sa * left_count + right_sa * right_count
     - Track best (axis, split_pos, cost)
  5. Check for straddling instances at the best split: instances whose bbox spans both sides
     - If straddling instances exist and there are enough non-straddling on each side: create NodeShare
     - Otherwise: create Node with the split and let straddling instances go to whichever side has more overlap
  6. Re-sort by best axis, split into left/right, recurse
  7. Return appropriate node type with combined bbox

NOTE: The NodeShare type is used when instances straddle the split plane. For simplicity in the initial implementation, if there are straddling instances, they go into the NodeShare's instance list. If all instances straddle (degenerate case), fall back to Leaf.

- `test_sah_split`: 8 items well-separated along X axis produce Node with children

- `test_surface_area`: SA of a unit cube is 6.0

  ```rust
      Mesh {
          instance: &'a RuntimeInstance,
          id: InstanceId,
      },
      Sprite {
          instance: &'a RuntimeInstance,
          id: InstanceId,
      },
  }
  ```
- `query_world_frustum` function:
  ```rust
  pub fn query_world_frustum(
      node: &BspNode,
      instances: &[RuntimeInstance],
      planes: &mut Vec<[f64; 4]>,
      callback: &mut impl FnMut(VisibleInstance),
  )
  ```
  Implementation (port of C++ RecurseWorldBSP):
  - Get node's bbox
  - Test bbox against frustum planes using same corner-counting logic as terrain:
    - Generate 8 corners from bbox
    - For each plane: count pos/neg corners
    - If neg == 8 for any plane: return (fully outside)
    - If pos == 8 for a plane: swap_remove (plane elimination)
  - CRITICAL: Clone planes before recursing into children (same Pitfall 5 as terrain)
  - For Node: recurse both children
  - For NodeShare: iterate straddling instances, check visibility flag, invoke callback for visible ones. Then recurse both children.
  - For Leaf: iterate instance list, check visibility flag, invoke callback
  - For Inst: check visibility flag, invoke callback
  - When invoking callback: check `instance.is_visible()`. If Mesh variant -> `VisibleInstance::Mesh`, if Sprite variant -> `VisibleInstance::Sprite`. Skip Item instances (not rendered in Phase 5).
