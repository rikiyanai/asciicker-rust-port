<objective>
Build the BSP tree runtime system that converts parsed A3dWorld instances into a spatial BSP tree with frustum-culled traversal dispatching mesh and sprite callbacks. Includes near-child-first traversal ordering for front-to-back rendering.

Purpose: The BSP tree is the spatial acceleration structure for world geometry. Frustum-culled BSP traversal during the WORLD stage determines which mesh and sprite instances are visible. Near-child-first ordering reduces overdraw by processing closer geometry first.

Output: `RuntimeWorld` resource with BSP tree built via SAH from parsed instances, frustum query with near-child-first ordering dispatching mesh/sprite callbacks, and instance flags controlling visibility.
</objective>

Create `bsp.rs` with:
- `BspNode` enum: Node, NodeShare, Leaf, Inst (all 4 C++ types)
  - `Node { children: [Option<Box<BspNode>>; 2], bbox: [f64; 6], split_plane: f64, split_axis: u8 }`
  - `NodeShare { children: [Option<Box<BspNode>>; 2], bbox: [f64; 6], instances: Vec<InstanceId> }`
  - `Leaf { bbox: [f64; 6], instances: Vec<InstanceId> }`
  - `Inst { bbox: [f64; 6], instance: InstanceId }`
- `BspItem { id: InstanceId, bbox: [f64; 6], centroid: [f64; 3] }`
- `build_bsp(items: &mut [BspItem]) -> BspNode`: SAH construction testing 3 axes
- `compute_surface_area`, `combined_bbox` helpers
- `VisibleInstance` enum for query callbacks (Mesh variant with InstanceId, Sprite variant with InstanceId)
- `query_world_frustum` with plane elimination, per-child plane cloning, and **near-child-first traversal**:
  - For `BspNode::Node`: determine which child is nearer to camera along split_axis
  - The function signature takes `camera_pos: [f64; 3]` parameter for ordering:
  ```rust
  // Near-child-first ordering (matches C++ RecurseWorldBSP):
  // Camera position determines which side of the split plane is "near".
  // Recurse near child first for front-to-back traversal, reducing overdraw.
  let near = if camera_pos[split_axis as usize] < split_plane { 0 } else { 1 };
  recurse(children[near], ...);   // near child first -> front-to-back
  recurse(children[1 - near], ...); // far child second
  ```
  - For `BspNode::NodeShare`: also process straddling instances between child recursions
  - For `BspNode::Leaf`/`BspNode::Inst`: dispatch directly to callback

Unit tests:
- `test_single_instance_bsp`, `test_small_set_bsp`, `test_sah_split`
- `test_combined_bbox`, `test_surface_area`
- `test_near_child_first_ordering`: Two instances on opposite sides of split plane. Verify callback fires for the instance nearer to camera position first.
- `test_node_share_straddling`: Instance that spans the split plane should appear in NodeShare
