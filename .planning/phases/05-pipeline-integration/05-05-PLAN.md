Create `assembly.rs` implementing the A3D-to-runtime assembly sequence. This resolves CRITICAL gap #2 and HIGH gap #5.

- `AssemblyState` resource tracking load state:
  ```rust
  #[derive(Resource, Default)]
  pub struct AssemblyState {
      pub a3d_handle: Option<Handle<A3dFile>>,
      pub assembled: bool,
  }
  ```

- `a3d_assembly_system` Bevy system:
  ```rust
  pub fn a3d_assembly_system(
      mut commands: Commands,
      mut assembly: ResMut<AssemblyState>,
      a3d_files: Res<Assets<A3dFile>>,
      terrains: Res<Assets<A3dTerrain>>,
      worlds: Res<Assets<A3dWorld>>,
      mat_tables: Res<Assets<MaterialTable>>,
      mut runtime_terrain: ResMut<RuntimeTerrain>,
      mut runtime_world: ResMut<RuntimeWorld>,
  )
  ```
  Implementation (port of C++ game.cpp load sequence):
  1. If `assembly.assembled` or `assembly.a3d_handle` is None, return early
  2. Check if the A3dFile asset is loaded via `a3d_files.get(handle)`
  3. If loaded, get its sub-asset handles for terrain, material table, and world
  4. Check all sub-assets are loaded
  5. When all ready:
     a. Build RuntimeTerrain: `runtime_terrain.build_from_parsed(terrain_asset)`
     b. Build RuntimeWorld: `runtime_world.build_from_parsed(world_asset)`
     c. Extract MaterialTable from the loaded asset and insert as a standalone Bevy Resource:
        ```rust
        let materials: Vec<Material> = convert_material_table(mat_table_asset);
        commands.insert_resource(RuntimeMaterials(materials));
        ```
     d. Trigger AKM mesh loads for each mesh referenced in world instances (queue handles)
     e. Compute terrain shadows via `update_terrain_dark` (from Plan 06)
     f. Set `assembly.assembled = true`
  6. Log info about what was assembled (patch count, instance count, material count)

- `RuntimeMaterials` wrapper resource:
  ```rust
  #[derive(Resource)]
  pub struct RuntimeMaterials(pub Vec<Material>);
  ```
  This is the standalone Bevy Resource wrapping the material table (resolving HIGH gap #5). The `Vec<Material>` uses the Phase 4 `Material` type, not the asset loader's raw MatCell arrays.

- `convert_material_table(table: &asset_loader::a3d_terrain::MaterialTable) -> Vec<Material>`:
  - For each of the 256 materials in the table:
    - Convert each `[[MatCell; 16]; 4]` entry to `render::material::Material`
    - The asset loader's MatCell and render's MatCell have the same layout (fg[3], gl, bg[3], flags = 8 bytes), so this is a field-by-field copy
  - Return Vec<Material> of length 256

- `load_a3d_scene` system (startup or trigger):
  ```rust
  pub fn load_a3d_scene(
      mut assembly: ResMut<AssemblyState>,
      asset_server: Res<AssetServer>,
  )
  ```
  - Loads the A3D file: `assembly.a3d_handle = Some(asset_server.load("game_map_y8.a3d"))`

- `render_pipeline_system`:
  ```rust
  pub fn render_pipeline_system(
      terrain: Res<RuntimeTerrain>,
      world_data: Res<RuntimeWorld>,
      camera: Res<GameCamera>,
      materials: Option<Res<RuntimeMaterials>>,
      config: Res<RenderConfig>,
      mut sample_buffer: ResMut<SampleBuffer>,
      mut cell_grid: ResMut<AsciiCellGrid>,
      mut timing: ResMut<PipelineTiming>,
  )
  ```
  Implementation following C++ Render() stage order with per-stage `Instant::now()` timing:

  **Stage 1: CLEAR**
  - `let t0 = Instant::now();`
  - `sample_buffer.clear()`
  - `timing.clear_us = t0.elapsed().as_micros() as u64;`

  **Stage 2: TERRAIN** (REAL rasterization, not placeholder)
  - `let t1 = Instant::now();`
  - If terrain has root and materials are available:
  - Get frustum planes from camera
  - `terrain.query_visible(planes, |patch, x, y| { render_patch(buf, w, h, patch, x, y, &camera.view_tm); })`
  - This calls TerrainShader from Plan 04 which calls Phase 4 rasterize()
  - `timing.terrain_us = t1.elapsed().as_micros() as u64;`

  **Stage 3: WORLD** (REAL rasterization for meshes, placeholder for sprites)
  - `let t2 = Instant::now();`
  - Create SpriteQueue
  - `world_data.query_visible(planes, camera_pos, |visible_inst| { ... })`
  - For Mesh: call `render_mesh(buf, w, h, mesh_data, &inst_tm, &camera.view_tm)`
  - For Sprite: compute distance from camera, project to screen, push SpriteRenderEntry
  - NOTE: Mesh rendering requires loaded AkmMesh data. If mesh not yet loaded, skip (log warning).
  - `timing.world_us = t2.elapsed().as_micros() as u64;`

  **Stage 4: SHADOW** (stub)
  - `let t3 = Instant::now();`
  - `// Stage 4: Player shadow -- deferred to Phase 6`
  - `timing.shadow_us = t3.elapsed().as_micros() as u64;`

  **Stage 5: REFLECTION** (stub)
  - `let t4 = Instant::now();`
  - `// Stage 5: Water reflection -- deferred to Phase 6`
  - `timing.reflection_us = t4.elapsed().as_micros() as u64;`

  **Stage 6: RESOLVE** (REAL resolve via bridge, not placeholder)
  - `let t5 = Instant::now();`
  - If materials available:
    - Call `resolve_to_grid(&sample_buffer, &materials.0, &mut cell_grid)`
    - This uses Phase 4 resolve() internally, then converts AnsiCell palette indices to AsciiCellGrid RGBA
  - `timing.resolve_us = t5.elapsed().as_micros() as u64;`

  **Post-RESOLVE: Deferred Sprite Blit**
  - `let t6 = Instant::now();`
  - `sprite_queue.sort_far_to_near()`
  - For each: `blit_sprite(&mut cell_grid, &entry, &sample_buffer)`
  - `timing.sprite_us = t6.elapsed().as_micros() as u64;`
  - `timing.total_us = (start of stage 1).elapsed().as_micros() as u64;`
