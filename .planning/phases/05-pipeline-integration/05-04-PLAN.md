<objective>
Wire the 6-stage rendering pipeline orchestrator system that connects terrain, world, camera, and output into a single Bevy system producing rendered ASCII output from scene data.

Purpose: This is the critical integration point where all Phase 5 subsystems converge. The pipeline orchestrator is the Rust equivalent of the C++ Render() function -- it executes the 6 stages in sequence, using frustum-culled terrain/world queries to populate the SampleBuffer, then resolves to AsciiCellGrid, then blits deferred sprites.

Output: `render_pipeline_system` Bevy system that runs each frame, producing rendered AsciiCellGrid output from RuntimeTerrain + RuntimeWorld + GameCamera + MaterialTable.
</objective>

  ```rust
  pub struct SpriteQueue {
      entries: Vec<SpriteRenderEntry>,
  }
  ```
- `SpriteQueue::new() -> Self` (empty queue)
- `SpriteQueue::push(&mut self, entry: SpriteRenderEntry)`: add sprite to queue
- `SpriteQueue::sort_far_to_near(&mut self)`: sort by dist descending (far first, near last -- painter's algorithm)
  - Use `entries.sort_by(|a, b| b.dist.partial_cmp(&a.dist).unwrap_or(Ordering::Equal))`
  - NOTE (Pitfall 7): C++ uses qsort (unstable). Using Rust's stable sort_by for deterministic order. If golden-file comparison requires matching C++ order exactly, switch to sort_unstable_by.
- `SpriteQueue::drain(&mut self) -> Vec<SpriteRenderEntry>`: take all entries, leaving queue empty
- `SpriteQueue::len(&self) -> usize`
