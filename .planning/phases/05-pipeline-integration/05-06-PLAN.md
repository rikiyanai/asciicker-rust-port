  <action>
Create `shadow.rs` implementing the C++ DarkUpdater equivalent. This is a load-time precomputation, NOT per-frame.

- `LIGHT_DIR_DEFAULT: [f64; 3]` constant:
  - Default light direction: `[-1.0, 1.0, -2.0]` (normalized, pointing from upper-right)
  - The C++ code computes light_dir from `light_pos` with Z scaled by HEIGHT_SCALE. We hardcode a reasonable default for Phase 5.

- `update_terrain_dark` function:
  ```rust
  pub fn update_terrain_dark(
      terrain: &mut RuntimeTerrain,
      light_dir: [f64; 3],
  )
  ```
  Implementation (port of C++ terrain.cpp:1714-1765 DarkUpdater):
  1. Two-pass approach to avoid borrow conflict:
     - **Pass 1**: Collect all shadow results. For each patch, iterate all 64 visual cells:
       a. Compute world-space cell center via `patch.sample_cell_center(u, v, px, py)`
       b. Cast a ray from cell center along `light_dir`
       c. Test terrain self-shadowing: `terrain_raycast_height` walks along ray, interpolating terrain height at each step
       d. If terrain height at step > ray height at step (with HEIGHT_SCALE/4.0 tolerance per C++), mark cell as shadowed
       e. Collect (patch_index, dark_bitmask) pairs
     - **Pass 2**: Write dark values to patches
  2. Write `patch.dark = dark` for each patch

  NOTE: Full world geometry shadowing (HitWorld) is deferred to Phase 6. Phase 5 implements terrain self-shadowing only. This is documented and acceptable because:
  - Terrain self-shadow is the dominant shadow contributor in most scenes
  - World shadow requires mesh raycast infrastructure not yet available
  - The dark bitmask format supports future world shadow additions (just OR in more bits)

- `terrain_raycast_height` helper:
  ```rust
  fn terrain_raycast_height(
      terrain: &RuntimeTerrain,
      origin: &[f64; 3],
      dir: &[f64; 3],
      max_steps: usize,
  ) -> bool
  ```
  Implementation:
  - Step along ray from origin in direction dir with step size = 1.0 (visual cell spacing)
  - At each step, compute world (x, y) position and interpolate terrain height via `terrain.interpolate_height(wx, wy)`
  - If interpolated terrain height > ray z at that step (with HEIGHT_SCALE/4.0 tolerance per C++), return true (shadowed)
  - After max_steps, return false (not shadowed)
  - max_steps default: 32 (sufficient for typical terrain extents)

- Shadow integration with assembly: The `a3d_assembly_system` in Plan 05 calls `update_terrain_dark` after building RuntimeTerrain. Add this call:
  ```rust
  // In assembly.rs after build_from_parsed:
  update_terrain_dark(&mut runtime_terrain, LIGHT_DIR_DEFAULT);
  ```

Update `terrain/mod.rs`:
- Add `pub mod shadow;`

Unit tests:
- `test_flat_terrain_no_shadows`: All-zero heights produces dark=0 for all patches
- `test_tall_peak_casts_shadow`: One elevated patch shadows adjacent lower patches in light direction
- `test_shadow_bitmask_layout`: Verify bit indexing: cell (u=3, v=5) maps to bit 3+8*5=43
- `test_shadow_is_load_time`: Verify update_terrain_dark is callable (not embedded in per-frame system)
