<objective>
Build the terrain quadtree runtime system that converts parsed A3dTerrain patches into a spatial quadtree structure with frustum-culled traversal.

Purpose: The terrain quadtree is the spatial data structure that enables efficient frustum culling during the TERRAIN stage of the 6-stage pipeline. Without it, every patch would be tested individually (O(n) vs O(log n)).

Output: `RuntimeTerrain` resource with quadtree built from parsed patches, frustum query with plane elimination, and RuntimePatch with height bounds and shadow bitmask slots.
</objective>

  - `visual: [[u16; 8]; 8]` (VISUAL_CELLS = 8 material cells per edge)

  - `dark: u64` (shadow bitmask, 64 bits for 8x8 cells, initially 0)
  - `lo: u16, hi: u16` (min/max height bounds, computed from height array)
- `RuntimePatch::from_terrain_patch(patch: &TerrainPatch) -> Self` constructor that:
  - Copies all fields from parsed TerrainPatch (from `crate::asset_loader::a3d_terrain::TerrainPatch`)
  - Computes `lo` and `hi` by scanning the 5x5 height array for min/max
  - Initializes `dark = 0`
- `RuntimePatch::sample_cell_center(&self, u: usize, v: usize, px: i32, py: i32) -> [f64; 3]` helper for shadow raycasting. Computes world-space coordinates of the center of visual cell (u,v) given patch world coords (px, py). Each patch spans VISUAL_CELLS in x/y, and each visual cell is 1 unit. The z coordinate is interpolated from the height grid using HEIGHT_CELLS/VISUAL_CELLS ratio. Use `crate::asset_loader::constants::{HEIGHT_CELLS, VISUAL_CELLS, HEIGHT_SCALE}`.

Create `quadtree.rs` with:
- `QuadNode` enum:
  ```rust
  pub enum QuadNode {
      Interior {
          children: [Option<Box<QuadNode>>; 4],
          lo: u16,
          hi: u16,
      },
      Leaf(RuntimePatch),
  }
  ```
- `QuadNode::height_bounds(&self) -> (u16, u16)` method returning (lo, hi) for both variants
- `build_quadtree(patches: &[TerrainPatch]) -> (Option<QuadNode>, i32, i32, i32)` function that:
  - Returns `(root, level, base_x, base_y)`
  - If patches is empty, returns (None, 0, 0, 0)
  - Determines the bounding box of all patch coordinates (min_x, max_x, min_y, max_y)
  - Computes the quadtree level as the smallest power-of-2 `range` such that `range >= max(max_x - base_x, max_y - base_y) + 1`
  - `base_x` and `base_y` are the minimum patch coordinates
  - Calls a recursive helper `insert_patches(patches, base_x, base_y, range, level)` that:
    - If level == 0 (leaf): finds the patch at (base_x, base_y), converts to RuntimePatch
    - Otherwise: partitions patches into 4 quadrants based on coordinates relative to base + half_range, recurses for each non-empty quadrant
    - Propagates lo/hi from children to Interior nodes (lo = min of children lo, hi = max of children hi)
  - Quadrant layout matching C++: children[0]=lower-left, children[1]=lower-right, children[2]=upper-left, children[3]=upper-right (or whichever convention the C++ uses -- the key is consistency)

Add to `quadtree.rs`:
- `FrustumResult` enum: `Outside, Inside, Partial`
- Helper `aabb_corners(bbox: &[f64; 6]) -> [[f64; 3]; 8]` that generates all 8 corners from [xmin, xmax, ymin, ymax, zmin, zmax]
- Helper `dot_plane(plane: &[f64; 4], point: &[f64; 3]) -> f64` returning `a*x + b*y + c*z + d` (plane is [a,b,c,d])
- `query_terrain_frustum` function:
  ```rust
  pub fn query_terrain_frustum(
      node: &QuadNode,
      x: i32, y: i32, range: i32,
      planes: &mut Vec<[f64; 4]>,
      callback: &mut impl FnMut(&RuntimePatch, i32, i32),
  )
  ```
  Implementation:
  - Get node's (lo, hi) height bounds
  - Construct AABB: `[x as f64, (x + range) as f64, y as f64, (y + range) as f64, lo as f64, hi as f64]`
  - Note: z-axis values should factor in HEIGHT_SCALE if needed for world-space matching. The C++ stores height as raw u16 values. Keep consistent with how the camera constructs frustum planes.
  - Test AABB against each frustum plane via corner counting:
    - Count corners with `dot_plane >= 0` (pos) and `< 0` (neg)
    - If neg == 8 for ANY plane: return immediately (fully outside, prune subtree)
    - If pos == 8 for a plane: remove that plane via `planes.swap_remove(i)` (plane elimination -- all descendants are inside this plane)
    - Otherwise: plane remains for child tests
  - CRITICAL (Pitfall 5): For Interior nodes, clone the planes Vec before passing to each child recursive call. The C++ uses a local copy. Without cloning, swap_remove in one child corrupts the plane set for siblings.
  - For Interior nodes: compute `half = range / 2`, offsets `[(0,0), (half,0), (0,half), (half,half)]`, recurse for each Some(child)
  - For Leaf nodes: invoke `callback(patch, x, y)`

- Import types: `use quadtree::{QuadNode, build_quadtree, query_terrain_frustum};`

- Define `RuntimeTerrain` resource:

  ```rust
  pub struct RuntimeTerrain {
      pub root: Option<QuadNode>,
      pub level: i32,
      pub base_x: i32,
      pub base_y: i32,
      pub patch_count: usize,
  }
  ```
- `RuntimeTerrain::build_from_parsed(terrain: &A3dTerrain)` method:
  - Calls `build_quadtree(&terrain.patches)` to get (root, level, base_x, base_y)
  - Sets all fields, `patch_count = terrain.patches.len()`
- `RuntimeTerrain::query_visible(&self, planes: &[[f64; 4]], callback: &mut impl FnMut(&RuntimePatch, i32, i32))`:
  - If root is None, return
  - Clone planes into a mutable Vec
  - Compute range from level: `range = 1 << self.level` (if level represents power of 2)
  - Call `query_terrain_frustum(&root, self.base_x, self.base_y, range, &mut planes_vec, callback)`
- `RuntimeTerrain::for_each_patch(&self, callback: &mut impl FnMut(&RuntimePatch, i32, i32))`:
  - Recursive walk of entire tree, no frustum test, invokes callback for every Leaf
- `RuntimeTerrain::for_each_patch_mut(&mut self, callback: &mut impl FnMut(&mut RuntimePatch, i32, i32))`:
  - Mutable recursive walk for shadow update (writes dark bitmask per patch)
- Register `RuntimeTerrain` as a resource in `TerrainPlugin::build()` via `app.init_resource::<RuntimeTerrain>()`
