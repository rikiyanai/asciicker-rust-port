SET 1: REFERENCE - HOW IT WORKS

1.1 XP File Structure

An .xp file contains 3 layers:

  • Layer 0: Color key (transparency) + Atlas metadata
  • Layer 1: Height encoding (0-35 per cell)
  • Layer 2: Visual drawing (glyph + fg/bg colors)

1.2 Layer 0 - Color Key + Metadata

Every cell: background color = transparency key

Metadata cells:

  • [0,0] = angle count (digit '1'-'8')
  • [heightx1, 0] = animation 1 frames
  • [heightx2, 0] = animation 2 frames
  • etc.

1.3 Layer 1 - Height

Each cell's glyph encodes Z-height for depth sorting:

  • '0'-'9' = heights 0-9
  • 'A'-'Z' = heights 10-35

1.4 Layer 2 - Visual

Each cell has: glyph (CP437) + foreground RGB + background RGB

Transparency: if fg/bk matches Layer 0's bk color = transparent

1.5 Atlas Layout

    Grid: columns = projs x sum(frames), rows = angles
    
    Example: 8 angles, 2 animations x 4 frames, projs=2
