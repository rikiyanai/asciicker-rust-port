# Engine Port Strategy Overview

## Status: REFERENCE (originally deferred in C++ project)

### Original Context (C++ Project)

The following decision rules and gate criteria were from the original C++ project's Python pipeline closeout. They are preserved here for historical reference but do not block the Rust port.

### Decision Rule (from original closeout plan)

1. Run real verification FIRST (no redesign).
2. Only if failures are real, apply minimal targeted fix.
3. If gates pass as-is: ship, no redesign.
4. Architecture cleanup is a separate RFC, frozen until gate evidence is complete.

### Gate Verification Criteria (from original closeout Step 4)

| Gate | Description | Criteria |
|------|-------------|----------|
| Gate 1 | Native Asciicker Conversion | angles=8, frames=[8], projs=2, layers>=3, XP size 256x160 |
| Gate 2 | Commercial Sprite Sheet | Metadata correct, visual export recognizable |
| macOS | Platform substitution | Metadata correct, viewer loads, visual export recognizable, 3+ layers, non-trivial glyphs, user sign-off |

## Port Strategy Summary

### Phase 0: Pipeline Verification (Historical -- not applicable to Rust port)

The original C++ closeout plan Steps 0-4 are not applicable to the Rust port.

| Outcome | Port Action |
|---------|-------------|
| Both PASS | Proceed to Phase 1 |
| PARTIAL | Document defects, reassess |
| FAIL | Apply closeout fixes, re-verify |

### Phase 1: Core Rendering Port

**Scope:** Translate C++ CPU rasterizer to Rust using Mage Core patterns.

**Duration Estimate:** 4-8 weeks

**Key deliverables:**
- Rust module structure matching Mage Core architecture
- SampleBuffer and AnsiCell types
- Template-based rasterizer (trait-based in Rust)
- 6-stage rendering pipeline

**Evidence required:**
- Render single-frame test scene
- Output matches C++ reference

### Phase 2: Asset Pipeline Integration

**Scope:** Integrate .xp sprite loading with Rust engine.

**Duration Estimate:** 2-4 weeks

**Key deliverables:**
- XP format parser (gzip + layer extraction)
- Sprite atlas assembly
- AnsiCell buffer population

**Evidence required:**
- Load `player-0100.xp` successfully
- Render sprite to screen

### Phase 3: Game Logic Migration

**Scope:** Migrate character system, input, inventory, networking.

**Duration Estimate:** 8-16 weeks

**Key deliverables:**
- Character/entity system
- Input handling
- Inventory management
- Networking (if applicable)

**Evidence required:**
- Playable demo with core mechanics
